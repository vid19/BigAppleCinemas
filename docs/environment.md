## GitHub Actions secrets for deploy workflow

### Staging (Environment: `staging`)

- `STAGING_MIGRATION_COMMAND`
- `STAGING_BACKEND_DEPLOY_COMMAND`
- `STAGING_FRONTEND_DEPLOY_COMMAND`

### Production (Environment: `production`)

- `PROD_MIGRATION_COMMAND`
- `PROD_BACKEND_DEPLOY_COMMAND`
- `PROD_FRONTEND_DEPLOY_COMMAND`

## Notes

- Keep production and staging secrets in separate GitHub Environments.
- Protect the `production` environment with manual approvals.
- Migration commands should be idempotent and safe to re-run.
